## Research Question

`VoltAgent/awesome-design-md` 是否值得作为 AI coding agent UI 设计规范、DESIGN.md 模板库、商业 UI 生成产品的参考对象？

## Bottom Line

`awesome-design-md` 是一组 DESIGN.md 文件集合，受 Google Stitch 的 DESIGN.md 概念启发，用纯 Markdown 描述 UI 风格，让 AI coding/design agents 生成一致界面。77K stars、MIT、73 个 DESIGN.md 样本。

**对 Claude_up 的核心价值**：`AGENTS.md`（构建规则）+ `DESIGN.md`（视觉规则）的分工模式，是 Claude_up 整个体系目前缺失的一层。`CLAUDE.md` 只负责行为和工程，没有视觉约束层——这是一个真实缺口，对杆杆响会员服务前端开发尤为重要。

## What To Borrow

| Pattern | Why It Matters | How To Adapt Locally | Confidence |
|---|---|---|---|
| DESIGN.md 作为设计代理上下文 | AI 读 Markdown 比读复杂设计文件更直接 | 每个前端项目根目录增加 DESIGN.md，描述布局、色彩、字体、组件、交互 | High |
| AGENTS.md + DESIGN.md 分工 | 一个管怎么构建，一个管看起来怎样 | Claude_up 项目中让 CLAUDE.md 负责工程行为，DESIGN.md 负责视觉系统 | High |
| 设计模板库 | 快速给 agent 一个视觉方向 | 在 Claude_up 09-templates/project/ 建 DESIGN.md 模板 | High |
| 品牌风格文本化 | 把"像某风格"转成结构化设计约束 | 为杆杆响自有品牌生成原创 DESIGN.md | Medium |

## Evidence

| Source | Type | Relevant Finding |
|---|---|---|
| GitHub 77K stars, MIT | 规模 | 高热度，活跃维护（2026-05-11 push） |
| README | 文档 | `AGENTS.md` = coding agents 读的构建规则；`DESIGN.md` = design agents 读的视觉规则 |
| Google Stitch 概念 | 官方 | plain-text design system document for AI agents |

## Applicability

**适用**：AI 生成前端页面需要一致设计方向；没有完整 Figma 但想约束 UI 风格；把品牌视觉规范给 Claude 读取
**不适用**：需要精确 Figma 1:1 实现；直接复刻知名品牌商业仿冒

## Risks And Limits

- Markdown 描述不能保证像素级一致，仍需视觉 QA
- 品牌风格、商标、字体本身可能有独立权利限制
